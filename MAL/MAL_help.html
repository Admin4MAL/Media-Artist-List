Introduction

My aim in developing Media/Artist Lists as a web application was to create something simple to use, that provides a quick and straight forward way of looking at a Squeezebox/Vortexbox based music collection. I wanted to be able to easily see all the material related to a specific artist in list format.

This project is made available under the GNU Public Licence and so is open for others either to contribute to or to clone and do their own thing with under the terms of that licence. I'm also happy to receive suggestions for additional features and improvements, though I offer no commitment as to if and when I will do the work as I'm also working on other projects.

The software is server based, it is installed on the LMS/Vortexbox server and is then accessible from all computers and mobile devices that have access to your network, without doing anything on the machine that uses it.

From an installation point of view, this application is just some extra files on your system, not making any changes to the Vortexbox/LMS configuration. The installation routine copies the files into the folder used by the web server and sets the security permissions on them that allows them to be used by the Web Server software. It works with its own copy of the LMS database file, not the one LMS uses as LMS prevents other applications from accessing it. Each time most pages load, they checks to see if the copy of the database file exists/has changed and copies it to its own location as required.

As the files sit within the folder from which the web server shares web pages and should work with any web server. I have tested it with Apache and the Nginx web server that Vortexbox currently uses. The application is fairly simple/generic it should work with other web servers.

I have targeted this for use on the Vortexbox systems, as I have been using it for some time and appreciates the benefits it brings. It should I guess work with any resent version of LMS that uses the SQLite database.

The information these pages display is only from the LMS database. LMS builds its database from information held within your music files and so will be as accurate/consistent as the music file tags/meta data. If you select an Artist Name or something similar and you have variations in the way that is stored within the database, then you will only get the variation you've selected, not the slight variations. Things like different abbreviations, capitalisation and different/miss-spelling and even extra spaces are all variations. These pages will work better the more consistent your information is. If the data is missing or different then the links can't be made!

I would also point out that I come from a world were music comes as Albums (or CDs), so that is my expectation within the application and its focus. If your focus is on tracks rather than Albums this might not work so well for you. Album lists aren't so useful if you aren't necessarily talking about complete Albums. You will have to remember that what shows as an Album might be one or more tracks from it, though looking at the Album detail will show the actual tracks you have. The Artist Track List should still be useful if you have a track focus.

Installation

The instructions for installing are found within the install.txt found within the zip file containing the application.

Starting the application

After you have carried out the installation, you start it by loading the MAL_start.php file in your web browser, such as Firefox or Chrome. To do this you need to type the IP address of your Vortexbox or LMS Server followed by a slash / and the file name MAL_start.php. This should be something like http://192.168.1.2/MAL_start.php, though the IP Address is very likely to be different. Pressing return or clicking on the reload the page icon in the browser once you have typed the address should load the page. When you are on this page you might like to bookmark/make the page a favourite, to avoid having to remember what you have just done, if you find the pages useful.

Page Layout/Usage

Every page presents you with a Title for the page, with the Main Menu (the same on every page) below it. The Main Menu includes options such as Help, returning to the Start Page and some other choices. On Most pages there is also a Page Specific Menu, that includes options that are specific to that page, including links to related pages and options to save to files. Some pages will have a Side Menu on the left hand side of the page, providing an option to jump to a point on the page where the listed items start with the number or letter selected on the menu. This is very useful on pages with long lists, being faster than scrolling.

You move around the pages using the Menus and some links within the pages. The links do not display as conventional web page links, but are consistent elements within the page and are coloured to reflect that. The Album Title, Artist Name and Track Title are all links. The Album Title always takes you to the Album Detail page for that album, the Artist name takes you to the Album list for that Artist and Track Title takes you to a list of occurrence of that Track. They are coloured differently to make them stand out. You can also use the web browser Back Button - [<-] to take you back to the previous page.

Pages within the Application

Start Page or Media/Artist List - Artist List

The Start Page on the menu, or Media/Artist List - Artist List as the page is titled, is the page that loads when you load MAL_start.php, it provides you with a list of all the Artists in your collection, divided in to Alphabetical, sections. You can move around the list using either the scroll bar or the side menu. Clicking on a Artist Name will take you to the Media/Artist List - Album List. This list includes all the Artists in your collection, whether there is are single tracks or complete albums by them. If it is possible the underlying LMS database has changed since you last loaded this page, it is worth reloading it to ensure the copy of the database is up to date.

Media/Artist List - Album Detail

The Media/Artist List - Album Detail provides you with the information about the album, which should include the cover art if it displays in LMS and a listing of all the tracks you have from it. If the Track Artists differ from the Album artist it should show the actual Artist and will also show variations in Genre. Clicking on an Artist Name will take you to the Media/Artist List - Album List for that Artist.

Media/Artist List - Artist Album List

The Media/Artist List - Artist Album List will provide a list of Albums/collections that includes tracks by the Artist Name you clicked on. This page focuses on the Albums/collections for an Artist, identifying the Albums based on the Album Name tag within the music file. It tries to group the Albums into groups that are explained below and orders the albums according to the Release Date tag. Clicking on the Album Name will bring up the Media/Artist List - Album Detail page.

On this page the Albums will be grouped in the following categories:

  1. Albums by the Artist.
  2. Albums on which the Artist is a guest.
  3. Various-Artist Albums on which the Artist appears.
  4. Albums on which the Artist composed Tracks.

Media/Artist List - Artist Composed List

When you are on a page that has a selected Artist you may see a button for the Artist Composed List on the Menu. The Menu only includes this button if the selected Artist has any tracks with the Artist marked as the Composer within the Tag/Meta-Data. If it appears, clicking on it should take you to a list of all the tracks Composed by that artist in your collection. The tracks are in Alphabetical order and the album they appear on is displayed in the list, clicking on that name will take you to the Media/Artist List - Album Detail page. This page is similar to the Artist Track List, but also shows the name of the Artist that recorded the track.

The Composer Tag/Meta-Data isn't used by a lot of people and would be more commonly used by Classical Music fans. Unless you have added this data yourself, you will probably find this page/list doesn't include everything it should and that the tagging in your collection is inconsistent.

Media/Artist List - Artist Track List

When you are on a page that has a selected Artist you will see Artist Track List on the Menu, clicking on this should take you to a list of all the tracks by that artist in your collection. The tracks are in Alphabetical order and the album they appear on is displayed in the list, clicking on that Album Name will take you to the Media/Artist List - Album Detail page.

Media/Artist List - Artist Responsible for:

This is the page that loads when you click on a Track, it presents you with all the instances of the Track you clicked on within the collection. It provides a list of the Artist who recorded the track, the Album on which they recorded it and the year in which it was released. Because Track Titles aren't necessarily unique these pages may list occurrence of tracks that are different, but that have the same title.

Media/Artist List - Full Album List

The Full Album List is a menu choice, that appears on every page, it takes you to a list of all the Albums/Collections in your collection ordered by Artist. It includes the Artist Name, that you can click on to get to the Media/Artist List - Album List page for that Artist and each album name you have, that will take you to the Media/Artist List - Album Detail page for that album. This list only includes artists you have Albums by, not Artists with tracks on albums by other Artists.

Media/Artist List - Full Track List

The Full Track List and Full Track List (Freq) are menu choices that appears on every page. The links take you to a list of all the Tracks within the Collection. Clicking on a Track Title takes you to the Artists Responsible for... page, that provides a list of all the occurrence of that Track. Because Track Titles aren't necessarily unique these pages may list occurrence of tracks that are different, but that have the same title.

The Full Track List version of this page is ordered by the Track Alphabetically and Full Track List (Freq) one is ordered by the number of times the Track occurs within the your collection. If you have a large music collection this these pages may take a while to load.

Printing

Printing pages is done using the standard Browser print functionality within the Browser menus. I have modified the styling of the pages (using Cascading Style Sheets) to make the pages look different when printed. The menus aren't included in the printouts and the font size is reduced to reduce the number of pages used for longer reports.

Some web browsers add a page header including the page title and URL and a footer with the page count and date. If you don't like including these you can remove or modify them within the printer settings as part of the Browser print settings.

If you wish to save a page to a file such as a PDF file you can do so (on Linux and some versions of Windows, or if you add additional software on other versions of Windows) by going to print the page and from the print dialogue box choosing to print to a PDF file.

CSV File

The CSV File button on the menu appears on pages that can be saved as CSV (Comma Separated Variable) format files. This provides a fairly straightforward way to get the data into another application where you can manipulate or format it.

CSV format files will load into spreadsheet applications (such as Libra Office Cal or MS Excel), probably by just clicking on the file and the application will provide you with a means to edit or format it as you wish.

Clicking on the CSV File Menu button will bring up a dialogue for you to download the file as you would any other file. The default file name suggested to you will be prefixed with the date in the format YYYYMMDD, which provides a means of versioning the file and sorting the files based on the name.

The data within the files isn't like the web pages where duplicated lines are left blank. In the CSV file every line has a complete set of data to provide more flexibility if you manipulate it.

Within the files, the variables/strings such as the Artist, Album and Track names are enclosed within quotation marks, this enables software that might open the file, to be able to differentiate between the commas that separate variables and those that appear within the names.

M3U Playlist

The M3U Playlist button appears on pages that include items that you might want to save as a playlist, where there are lists of tracks. It doesn't appear on the Full Track List pages as the output from these is too large and LMS has a Random playlist function that would achieve the same.

Possible Playlists:

  1. Album track list - ordered list.
  2. Artists Responsible for - Randomised.
  3. Tracks Composed by Artist - Randomised.
  4. Tracks Performed by Artist - Randomised.

The Artists Responsible for List give you a playlist of all the tracks with the same name, which is interesting for tracks that have been frequently covered or for older tracks, most likely found in the Blues and Folk genres where standard tracks are performed by different artists with a lot of variation.

The playlist is saved to your browsers download location, so you have to copy it to the LMS playlist folder and select to reload all the playlists within LMS before it's available.

Future development/features of these pages

I developed this to provides me with information I wanted to see that isn't easily viewable in LMS. I have made it Open Source because as a Linux user I believe in that as a principle, but also felt it might be useful to others saving them work and giving them opportunity to develop it further if they wish. In the spirit of Open Source I'm happy for others to contribute and improve. I have other features that I may add, but I'm also open to requests for others features, provided they can work within the constraints of the LMS database and the tools in the toolkit (HTML, PHP, CSS and SQLite).

Customisation

The application provides a framework that is fairly easy to modify, because everything is in plain text files that are easy to edit using a text editor. Though if you change something and it doesn't work, it can be difficult tracking down the problem, so I would suggest the change a little and test approach rather than making big changes.

If you are making changes, particularly if you have already made changes, remember to back them up. If you haven't changed anything you can just re-install.

CSS - Changing the way it looks

I have put the CSS in a separate file 'MAL.css', which is read when all the files are created. This means that modifying the cosmetic aspects of the application can be done just by modifying this file, if your plans for modification aren't too great. I have commented the tag styling within the file to help with this.

SQL - Changing the data it displays

The SQL is embedded within the PHP and is used to read the data from the database. If you feel you can improve on or wish to change the way the data is read or what data is read you can update the SQL within these statements. If you read additional or different fields from the database you may also need to modify the PHP to get that data displayed.

When working on the SQL I copied the database to another machine and used a Firefox add-on called 'SQLite Manager', which lets you open the database and run SQL against it, to develop and test the code, before pasting it into the PHP statements.

PHP - Changing the way it works

As PHP is an interpreted language, it is read and acted upon as the page loaded, so any changes you make will be applied next time you load the page after you have saved it. Unfortunately if it doesn't work, it just doesn't work not giving you too much in the way of clues as to what is wrong.

Missing a ';' off the end of a line of code frequently causes a page not to work, without giving any clues, so it's worth checking you haven't done this if a page doesn't load. If you are going through steps to achieve something it is worth adding each step and then testing before adding the next and also including echo statements that state where they are, so you know how far the code is getting when it runs.

Glossary

ApacheApache is a freely available Web server that is distributed under an "open source" license. It runs on most most operating systems and is probably the most widely used Web Server on the Internet.
CSSCascading Style Sheet, the markup language used for applying graphical style to Web Pages.
CSVComma Separated Variable, a text based file format used for transfering data between different software.
FLACFree Lossless Audio Codec - A Lossless audio encoding format.
GNUA recursive acronym for "GNU's Not Unix."
HTMLHyper Text Mark-up Language, the language used for laying out Web Pages.
LMSLogitech Media Server, also know and the Squeezebox server. This server is now Open Sourced (under GNU General Public License).
M3UA text based file format for media playlists.
Nginx(pronounced "engine x") A freely available Web Server.
PHP"PHP: Hypertext Pre-processor" (A recursive acronym). A scripting language used within HTML, that runs of the server dynamically building web pages that appear on the client.
SQLStructured Query Language - A programming language used for managing and querying relational databases.
SQLiteAn in-process library that implements a self-contained, serverless, zero-configuration, transactional SQL database engine.
SqueezeboxSqueezebox is a network music player from Logitech that uses LMS as its server.
VortexBoxVortexBox is a free, open source (GPL v3), Linux distribution that provides a ready-made and easy to use music server/jukebox. Its core is the LMS/Squeezebox system.
